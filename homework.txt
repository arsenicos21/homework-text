Поступает команда по подгрузке класса JvmComprehension
Начинается работа подсистемы загрузчиков классов
Приходит запрос в Application ClassLoader, который передает запрос загрузчику более низкого уровня - Platform ClassLoader, который, в свою очередь, передает запрос загрузчику Bootstrap ClassLoader
Далее начинается работа по поиску класса загрузчиками от самого нижнего уровня
Если загрузчик находит класс, то он его загружает, в ином случае загрузчик более низкого уровня возвращает запрос на поиск класса загрузчику более высокого уровня
Если ни в одном из загрузчиков класс не находится, будет выдана ошибка ClassNotFoundException
В метаспейсе хранятся данные о классе, константы
Загружается вызванный класс JvmComprehension из метаспейса, JVM ищет точку входа - метод main(), и вызывает его, начинается выполнение программы
В момент вызова метода main() создается фрейм в стековой памяти
1. Создается целочисленная переменная типа int(Integer) i, равная 1
2. Создается объект Object в куче (heap), а ссылка на него (o) находится в стеке
3. Создается целочисленная переменная типа int(Integer) ii, равная 2
4. Идет вызов метода printAll(), создается новый фрейм в стеке, куда будет передана ссылка на объект o и целочисленные переменные i и ii
5. Создается целочисленная переменная типа int(Integer) uselessVar, равная 700
6. Создается еще один фрейм в стеке, куда передается ссылка на объекты метода System.out.println() - ссылка для toString, который вызывается отдельным фреймом в стеке для объекта o, и целочисленные переменные i и ii
Стек начинает очищаться, фрейм для метода printAll() будет удален
7. Создается фрейм в стеке, куда передается строка "finished". По выполнению метода System.out.println() метод main() заканчивает свое выполнение, стек очищается и становится пустым
Программа завершит свою работу
